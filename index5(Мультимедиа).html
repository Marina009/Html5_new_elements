Мультимедиа

Вступление

Когда сеть только зарождалась, это была простая тектстовая среда, которая состояла в основном из списков, абзацев и гиперссылок. И примерно в 1993 году на рынке появились первые графические веб-браузеры, в которых визуальные элементы, такие как изображения и графика, заняли центральное место.
А роль видео- и аудиоматериалов в Интернете сильно возросла с появлением HTML5. И, благодаря интеграции мультимедиа в HTML5, веб-разработчики смогли включать информацию в формате мультимедиа в свои проекты, не нуждаясь в поддержке третьих сторон (например, как это было с Flash).

HTML5 представил два новых элемента: audio и video, которые включают функции воспроизведения для поддерживаемых форматов мультимедиа. А также еще появились три элемента, которые помогают настраивать мультимедийный
контент – это embed, source и track.

Рассмотрим же теперь эти элементы подробнее!

Тег <video>

Синтаксис:

<video>...</video>


О теге:

Встраивает видео на веб-страницу. Адрес видео задаётся внутри контейнера <video> с помощью тега <source> или атрибута src. Любой текст внутри элемента <video> отображается браузером в том случае, когда сам элемент или формат используемого видеофайла не поддерживается браузером.

Также, если браузер не поддерживает определённый формат видео, можно добавить в тег несколько форматов одного и того же файла. На данный момент для видеоэлемента поддерживаются три основных видеоформата: MP4, Ogg и WebM.


Разрешённый контент тега:

Текстовый контент; если данный элемент имеет атрибут src, то ноль или более элементов track; если элемент не имеет атрибута src, то ноль или более элементов source, ноль или более элементов track.

Специальные атрибуты
Атрибут Значение Описание
autoplay autoplay Автоматическое воспроизведение мультимедийного файла после загрузки страницы.
controls controls Добавляет стандартные элементы управления видеоплеером: кнопку воспроизведения и паузы, полосу прокрутки, громкость звука, полноэкранный режим и другие элементы, в зависимости от браузера.
height пиксели Задаёт высоту окна видеоплеера.
loop loop Указывает, что воспроизведение будет начинаться сначала, каждый раз, после завершения видео.
muted muted Видео начнёт играть без звука, пока его не включит пользователь.
poster URL URL-адрес картинки, которая будет показываться, пока видео не загрузится. Если её не будет, то, пока не загрузится первый кадр, на месте видео пользователь увидит чёрный прямоугольник.
preload none

metadata

auto		 Указывает, какие действия должен выполнять браузер, если для видеоплеера не указан атрибут autoplay. При совместном использовании с атрибутом autoplay атрибут preload игнорируется. Значение none означает, что загрузка видеофайла начнется только после щелчка мыши по кнопке воспроизведение. Metadata осуществляет сбор информации о видеофайле: размер, первый кадр, список воспроизведения, длительность. С помощью значения auto загрузка видеофайла начинается после загрузки страницы.
src URL Определяет путь к нужному видеофайлу (абсолютный или относительный).
width пиксели Устанавливает ширину видеоплеера.

Пример применения тега

            <video controls="controls" src="video.mp4" width="320px">
    Ваш браузер не поддерживает это видео.
</video>

Тег <source>

Синтаксис:

<source/>


О теге:

Используется внутри элементов audio и video для указания расположения файлов. Также можно использовать несколько элементов <source> для указания нескольких альтернативных аудио/видео файлов, чтобы обозначить, что они доступны в нескольких форматах. Браузер может просматривать список элементов source до тех пор, пока не найдёт формат, который можно воспроизводить.

Пример данного тега находится ниже, в комбинации с элементом audio .


Разрешённый контент тега:

Пустой, так как этот элемент включает только начальный тег и не может содержать какие-либо контенты.

Специальные атрибуты
Атрибут Значение Описание
src URL Определяет путь к нужному видеофайлу (абсолютный или относительный).
media медиа-запросы Указывает под какие типы устройств оптимизирован файл. Браузеры могут использовать этот атрибут для определения, возможно ли воспроизведение файла или нет.
type тип медиа Определяет MIME-тип медиа ресурса. Возможные значения атрибута для видео-файлов: video/ogg; video/mp4; video/webm, а для звуковых файлов: audio/ogg; audio/mpeg.

Тег <audio>

Синтаксис:

<audio>...</audio>


О теге:

Добавляет на страницу аудиоплеер, который воспроизводит звуковой файл. Можно также добавить несколько форматов одного файла, чтобы браузер воспроизвёл тот формат, который он поддерживает — для этого нужно добавить вложенные теги <source>.


Разрешённый контент тега:

Включающий атрибут src либо один или более элементов source, за которыми следует потоковый или фразообразующий контент. Применение элементов video и audio не допускаются.

Специальные атрибуты
Атрибут Значение Описание
autoplay autoplay Автоматическое воспроизведение аудиофайла после загрузки страницы.
controls controls Добавляет стандартные элементы управления аудиоплеером: кнопку воспроизведения и паузы, громкость звука и другие элементы, в зависимости от браузера.
loop loop Указывает, что воспроизведение будет начинаться сначала, каждый раз, после завершения аудио.
muted muted Звук будет на нуле, пока пользователь его не увеличит.
preload none

metadata

auto		 Указывает, какие действия должен выполнять браузер, если для аудиооплеера не указан атрибут autoplay. При совместном использовании с атрибутом autoplay атрибут preload игнорируется. Значение none означает, что загрузка аудиофайла начнется только после щелчка мыши по кнопке воспроизведение. Metadata осуществляет сбор информации об аудиофайле: размер, список воспроизведения, длительность. С помощью значения auto загрузка аудиофайла начинается после загрузки страницы.
src URL Определяет путь к нужному аудиофайлу (абсолютный или относительный).

Пример применения тега

            <audio controls="controls">
    <source src="beautiful_song.mp3" type="audio/mp3">
    Ваш браузер не поддерживает эту аудиозапись.
</audio>

Тег <embed>

Синтаксис:

<embed/>


О теге:

Используется для вставки на страницу контента в сторонних форматах, которые браузер отображает с помощью плагинов. Чаще всего используется для видео- и аудиоконтента.

Если на компьютере пользователя не установлен нужный для отображения контента плагин, то он увидит в окне предупреждение, например: "Плагин Adobe Flash Player больше не поддерживается".


Разрешённый контент тега:

Пустой, так как этот элемент включает только начальный тег и не может содержать какие-либо контенты.

Специальные атрибуты
Атрибут Значение Описание
src URL Указывает адрес внешнего файла, который будет вставлен в элемент.
type тип медиа Определяет тип загружаемого контента в формате MIME. Тип указывать не обязательно, но желательно. Так браузер будет точно знать, какой плагин использовать для отображения контента.
width пиксели Определяет ширину вложенного содержимого.
height пиксели Определяет высоту вложенного содержимого.

Пример применения тега

            <embed src="video3.mov" width="320px" type="video/quicktime">

Тег <track>

Синтаксис:

<track/>


О теге:

Используется в качестве дочернего элемента для тегов <audio> и <video> для указания субтитров, заголовка файла или других файлов, содержащих текст, которые должны быть видны при воспроизведении.


Разрешённый контент тега:

Пустой, так как этот элемент включает только начальный тег и не может содержать какие-либо контенты.

Специальные атрибуты
Атрибут Значение Описание
src URL Указывает URL-адрес файла с дорожкой субтитров. Является обязательным атрибутом.
default default Указывает, что дорожка должна быть включена, если пользовательские настройки это позволяют.
kind
captions

chapters

descriptions

metadata

subtitles

 Определяет тип текстовой дорожки, если атрибут не указан, то по умолчанию используется значение subtitles, которое обеспечивает транскрипцию или перевод диалога, который отображается в окне воспроизведения видео. Значение captions обеспечивает полное описание аудиофайла, включая звуковые эффекты и другие звуки, когда звук недоступен, или для людей со слабым слухом. Значение chapters обозначает заголовки глав, используемых для навигации по мультимедийному ресурсу. Descriptions указывает текстовое описание видео контента, например, для слепых пользователей. Metadata представляет дорожку, которая предназначена для использования сценариями (эта информация не видна для пользователей).
label текстовая строка Указывает заголовок для дорожки, используемый браузером при перечислении доступных текстовых дорожек.
srclang код языка Указывает двухбуквенный код языка, который используется в текстовой дорожке.
Этот атрибут обязатален, если kind="subtitles".

Пример применения тега

            <video controls="controls" src="video.mp4" width="320px">
    <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
</video>
